Introduction

In this project, I will create a responsive redesign of the website When2meet. I will identify the learnability, usability, memorability, and accessibility issues present in the original webpage, create low-fidelity and high-fidelity wireframes, and finally create a responsive webpage for my redesign. The redesign will focus on user-centered design, accessibility, and responsiveness.

Part 1: Identifying User Problems

Picking a Web Page

When2meet is simple and effective. It solves one problem and one problem only: scheduling an event at a time when all participants are available. Although I love the simplicity of the website, the user interface, learnability, and efficiency can definitely be improved. I will be focusing on the "Plan a New Event" page.

When2meet

Finding Problems

learnability

  • hard to learn
  • overwheling with information and visual noise
  • makes user look at all the steps of making a new event at once (this includes selecting event date, selecting event times, and selecting event name)
  • scrolling on calendar is incredibly unintuitive, no visible scroll bar

usability

  • the labeling of the calendar is confusing
  • creating a one time event or a recurring event is confusing
  • dropdown menus for times are confusing

memorability

  • fairly easy to master with continuous use

miscellaneous

  • not visually appealing

Accessibility

  • language of page is not specified
  • "New Event Name" box does not have an alternative lable for screen reader users
  • navigation bar links do not have alternative labels for screen reader and keyboard users
